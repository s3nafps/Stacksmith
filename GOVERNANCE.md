# Governance

Stacksmith starts with a lightweight maintainer-led model.

Maintainers are responsible for roadmap direction, issue triage, pull-request review, releases, security handling, and final technical decisions.

Changes are normally accepted through pull requests. Maintainers may request tests, documentation, or security clarifications before merging. Release authority stays with maintainers until delegated.

Future maintainers may be nominated based on sustained, high-quality contributions and trust with project security boundaries.

Disagreements should be resolved in public project threads where possible. Maintainers make final calls when consensus is not reached.
